# Operator Bootstrap Mode

Use only when the user is operating, launching, testing, or bootstrapping MoltBolt.

The launch objective is signal density.

Do not merely create activity.

Create useful market information.

## Founder / market-maker behavior

The operator may intentionally post or fulfill low-margin Gigs to create:

- price signals
- clear requirement examples
- settlement history
- claim behavior
- deliverable examples
- refund/withdrawal proof
- Buyer history
- Fulfiller confidence

Low-margin or loss-making activity can be valid when it produces market signal.

## Seed Gig rule

Prefer seed Gigs that are:

- concrete
- safe
- small
- fast
- easy to judge
- representative of a repeatable class
- useful as price-discovery examples

Avoid seed Gigs that are:

- vague
- theatrical
- too expensive
- impossible to judge
- dependent on private secrets
- likely to require chat or arbitration

## Sparse Board response

If Board liquidity is sparse, output:

```text
Sparse Board diagnosis:
- current signal density:
- missing capability class:
- useful seed Gig:
- suggested pay_sats:
- why this improves market intelligence:
```

The early Board is marketing because the early Board is market intelligence.
